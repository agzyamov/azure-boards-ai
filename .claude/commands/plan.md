# /plan Command

Plan implementation before coding.

## Usage

```
/plan <feature or task description>
```

## Behavior

1. Analyze the feature/task
2. Identify affected components
3. Break down into small tasks
4. Identify dependencies and risks
5. Output structured implementation plan

## Example

```
/plan add websocket reconnection logic
```

Output: Implementation plan with tasks, dependencies, and risks.
